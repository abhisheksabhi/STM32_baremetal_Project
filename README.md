# STM32F4 Bare-Metal Round-Robin Task Scheduler

**Target:** STM32F407 (Cortex-M4, HSI 16 MHz)

## Overview

This project implements a cooperative/preemptive round-robin task scheduler from scratch on the STM32F4 series, without any RTOS or HAL library. It demonstrates bare-metal startup, a custom linker script, SysTick-driven context switching via PendSV, and GPIO-based LED control.

Four user tasks blink four on-board LEDs (STM32F4-Discovery: PD12‚ÄìPD15) at different rates. An idle task runs when all user tasks are blocked.

## Startup Sequence (`stm32_startupByMe.c`)

1. Vector table placed at `0x08000000` (`.isr_vector`) ‚Äî first entry is `STACK_START`.
2. `Reset_Handler` runs on power-on/reset:
   - Copies `.data` initializers from FLASH (`_loadaddrss_data`) to SRAM (`_sdata`) ‚Äî word-based copy (`uint32_t`), 4-byte alignment guaranteed by linker.
   - Zero-initializes `.bss` (`_sbss` to `_ebss`), word-based.
   - Calls `__libc_init_array()` for C runtime / C++ constructors.
   - Calls `main()`. If `main()` returns, loops forever.
3. All IRQ handlers are weakly aliased to `Default_Handler` (infinite loop).  
   Overridden in `main.c`: `HardFault_Handler`, `MemManage_Handler`, `BusFault_Handler`, `PendSV_Handler`, `SysTick_Handler`.

## Scheduler Design (`main.c`)

- `MAX_TASKS = 5` (1 idle + 4 user tasks).
- **TCB** (Task Control Block) holds: `psp_value`, `block_count`, `current_state`, `task_handler`.
- Task states: `TASK_READY_STATE` (`0x00`), `TASK_BLOCKED_STATE` (`0xFF`).

### Initialization sequence

1. `enable_processor_faults()` ‚Äî enables MemManage, BusFault, UsageFault via `SCB->SHCSR`.
2. `initialise_monitor_handles()` ‚Äî semihosting init (printf output via debugger).
3. `init_scheduler_stack()` ‚Äî sets MSP to `SCHED_STACK_START` (naked function, `MSR MSP`).
4. `init_tasks_stack()` ‚Äî fills each task's initial stack frame:
   - `xPSR = 0x01000000` (Thumb bit set)
   - `PC` = task handler address
   - `LR = 0xFFFFFFFD` (EXC_RETURN: return to Thread mode, use PSP)
   - `R0‚ÄìR12 = 0`
5. `led_init_all()` ‚Äî enables GPIOD clock, sets PD12‚ÄìPD15 to output mode.
6. `init_systick_timer(1000)` ‚Äî 1 ms tick at 16 MHz HSI (count = 15999).
7. `switch_sp_to_psp()` ‚Äî loads PSP with task1's stack, sets `CONTROL.SPSEL = 1`.
8. Enters `task1_handler()` directly.

### Context switching (`PendSV_Handler`, naked)

- **Save:** `MRS R0,PSP` ‚Üí `STMDB R0!,{R4-R11}` ‚Üí save R0 to current TCB.
- `update_next_task()`: round-robin; skips `TASK_BLOCKED_STATE` tasks; falls back to idle.
- **Restore:** load next TCB's PSP ‚Üí `LDMIA R0!,{R4-R11}` ‚Üí `MSR PSP,R0`.
- `BX LR` ‚Äî EXC_RETURN `0xFFFFFFFD` restores `xPSR/PC/LR/R0‚ÄìR3/R12` from stack.

### `SysTick_Handler` (1 ms)

- Increments `g_tick_count`.
- `unblock_tasks()`: marks tasks READY when `block_count == g_tick_count`.
- Pends PendSV for context switch.

### `task_delay(tick_count)`

Disables interrupts (PRIMASK), sets `block_count = g_tick_count + tick_count`, sets state to `TASK_BLOCKED_STATE`, pends PendSV, re-enables interrupts.

## Semihosting / printf

`syscalls.c` provides Newlib stubs. `initialise_monitor_handles()` is a no-op stub. To enable actual semihosting output via a debugger, link with `--specs=rdimon.specs` and implement `__io_putchar` / `__io_getchar`, or connect a UART and redirect `_write` there.

## Known Issues

### Compile error in `switch_sp_to_psp()` (`main.c`)

There is a stray `e` in one inline asm statement:

```c
__asm volatile ("PUSH {LR}"e);   // <-- stray 'e', causes compile error
```

Fix:

```c
__asm volatile ("PUSH {LR}");
```

### FPU not initialized

The `#warning` at the top of `main.c` fires when compiling for hard-FP. Either initialize the FPU in `Reset_Handler` before calling `main()`, or use `-mfloat-abi=soft`.

### INTERRUPT_DISABLE / INTERRUPT_ENABLE clobber R0

The macros in `main.h` use `MOV R0` + `MSR PRIMASK` without declaring R0 as clobbered. A safer alternative:

```c
__asm volatile ("CPSID I" ::: "memory");  // disable
__asm volatile ("CPSIE I" ::: "memory");  // enable
```
